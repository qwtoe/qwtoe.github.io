基本git操作

Git 常用的是以下 6 个命令：git clone、git push、git add 、git commit、git checkout、git pull，后面我们会详细介绍。

说明：

  • workspace：工作区
  • staging area：暂存区/缓存区
  • local repository：版本库或本地仓库
  • remote repository：远程仓库

一个简单的操作步骤：

  • git init - 初始化仓库。
  • git add . - 添加文件到暂存区。（“add” 和 “.” 之间有空格）
  • git commit - 将暂存区内容添加到仓库中。

提交和修改

命令 说明
git add 添加文件到暂存区
git status 查看仓库当前的状态，显示有变更的文件。
git diff 比较文件的不同，即暂存区和工作区的差异。
git commit 提交暂存区到本地仓库。
git reset 回退版本。
git rm 将文件从暂存区和工作区中删除。
git mv 移动或重命名工作区文件。

2022-11-09 更新

  1. 由于 HEAD 指针的回退，导致有的 commit 在 git log 中看不到，这时候用 git reflog

  2. 将缓存区内容恢复到工作目录：git checkout a.txt 、git chechout . 恢复之后无法撤回，谨慎使用

  3. commit 之后，发现需要修改错别字，又不想新开一个 commit ，这时候用 git commit --amend ，这样会将错别字的修改和之前的 commit 中的修改合并为一个 commit 提交。

  4. 使用 git add . 之后，发现 a.txt 还需要编辑，想把它从 缓存区 撤销，这时候使用 git reset a.txt ，此时 缓存区 已经没有 a.txt 了，commit 也不会提交 a.txt

  5. git add . 和 git commit 可以合并为 git commit -a

  6. 恢复成最初的样子：git checkout HEAD .

2022-11-10 更新

  1. 放弃本地修改，强制拉取更新：git reset --hard origin/master
